Environment & Config
12-factor principles, secrets vs config, feature flags, and how to ship the same artifact through dev, staging, and prod without rebuilding.
Twelve-Factor App (summary)
Codebase · Dependencies · Config
One repo per app. Explicit dependency declaration (go.mod, package-lock). Config in environment — never committed secrets. Build produces one immutable artifact.
Backing services · Build/release/run
Treat DB/cache as attached resources (URLs in env). Strict separation: build = compile, release = build + config, run = execute. Never SSH to change prod.
Processes · Port binding · Concurrency
Stateless processes; sticky sessions are a smell. Export HTTP on $PORT. Scale out with process model (workers), not threads in one giant process.
Disposability · Dev/prod parity · Logs
Fast startup, graceful SIGTERM. Minimize drift: same Postgres major in dev as prod. Stream logs to stdout — platform aggregates.
Admin processes · One-off tasks
Migrations and console jobs run as one-off processes with same release as app — same codebase, same config version.

// SECRET SCANNING

Enable GitHub secret scanning, gitleaks, or trufflehog in CI. A leaked key in history is forever — rotate immediately and use scoped keys per service.

Feature Flags
Release vs experiment
Release flags hide incomplete work; remove after launch. Experiment flags drive A/B tests with metrics — owned by product/analytics.
Kill switches
Flags that turn off expensive paths (third-party API, new code path) without deploy. Must be observable — log flag state on critical paths.

Testing Strategy
The test pyramid, contract tests, property-based testing, and where to invest so failures happen before production — not after.
Test Pyramid
UnitMany, fast, isolated. Mock external I/O. Cover pure logic and edge cases. Run on every commit.70%
IntegrationReal DB, real queue, real HTTP — but in-process or docker-compose. Catch wiring bugs. Slower than unit.20%
E2E / UIFull browser or API black-box. Flaky, expensive. Critical user journeys only. Run nightly or pre-release.10%
// ICE CREAM CONE ANTI-PATTERN

Most tests are E2E, few unit tests — slow CI, brittle feedback. Invert: push confidence down the pyramid.

Test Types at a Glance
TypeValidatesTools

SnapshotUI or JSON output unchanged (guard against drift)Jest, Vitest — review diffs carefully
Property-basedInvariants hold for random inputsHypothesis, fast-check, QuickCheck

Determinism & Flakes
Common flake sources
Time (now()), random IDs, async without await, shared global state, order-dependent assertions, real network in unit tests.
Fixes
Inject clocks; seed RNG; fake timers; retry only at E2E layer with cap; quarantine flaky tests until fixed — never ignore permanently.

Performance
Latency vs throughput, profiling, caching layers, and the discipline of measuring before optimizing — because intuition lies.
Latency Budget

If p99 API SLO is 200ms and you have 5 hops (LB, gateway, service, DB, cache), each hop gets a slice — often ~20–40ms max. Serial chains multiply; parallelize independent work.

Little's Law
L = λ × W — average items in system = arrival rate × average time in system. Queue depth explodes when utilization → 100%.
Tail latency
p99 matters more than mean. One slow dependency in a fan-out poisons the whole request — hedging, timeouts, bulkhead isolation.

// 15 · engineering
Observability
The three pillars used correctly: metrics for aggregates, logs for context, traces for causality — plus SLOs and actionable alerts.
Three Pillars
Metrics
Counters, gauges, histograms. RED: Rate, Errors, Duration per service. Cardinality explosion = death — avoid high-cardinality labels in Prometheus.
Logs
Structured JSON with trace_id, level, msg. Correlate with traces. Sampling for high volume.
Traces
Distributed spans with parent span ID. Shows which downstream call burned latency. W3C trace context propagation across HTTP/gRPC.
SLO / SLI / SLA
SLI
What you measure — e.g. ratio of successful HTTP 2xx / total requests over 30 days.
SLO
Target — e.g. 99.9% availability monthly. Error budget = allowed downtime — spend on velocity or freeze releases.
SLA
Contract with customers — credits or penalties if breached. Stricter than internal SLO.
